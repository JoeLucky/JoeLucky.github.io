    再进行解压缩，此时npm是已经链接到npm-cli.js的。

screen相关 下载

1、干嘛用的：
    系统管理员经常需要SSH 或者telent 远程登录到Linux 服务器，经常运行一些需要很长时间才能完成的任务，比如系统备份、ftp 传输等等。通常情况下我们都是为每一个这样的任务开一个远程终端窗口，因为它们执行的时间太长了。必须等待它们执行完毕，在此期间不能关掉窗口或者断开连接，否则这个任务就会被杀掉，一切半途而废了。
    上面一段是复制的，简言之，screen最基本的功能是远程登录linux，启动程序X，关闭远程连接，程序X不停止。
2、基本用法：
    screen -S test1  新建
    Ctrl+a+d                    #按Ctrl+a，然后再按d即可保留当前的screen会话
    exit                        #这样就表示成功退出了
    ////////////////////////
    screen -ls       #查看所有screen会话  #假设显示 9649.test1   (Detached) Deatched表示没连接
    screen -r test1 (or 9649)
    如果上一条命令不能进入指定screen 如提示
    screen -d test1 将指定的screen作业离线
    screen -wipe  test1   #检查所有作业 删除已经无法使用的作业

Linux防火墙问题

有时候必须关闭防火墙才能操作某些东西 怎么关闭？如下
1) 重启后生效 
开启： chkconfig iptables on 
关闭： chkconfig iptables off 

2) 即时生效，重启后失效 
开启： service iptables start 
关闭： service iptables stop 

需要说明的是对于Linux下的其它服务都可以用以上命令执行开启和关闭操作。 

在开启了防火墙时，做如下设置，开启相关端口， 
修改/etc/sysconfig/iptables 文件，添加以下内容： 
-A RH-Firewall-1-INPUT -m state --state NEW -m tcp -p tcp --dport 80 -j ACCEPT 
-A RH-Firewall-1-INPUT -m state --state NEW -m tcp -p tcp --dport 22 -j ACCEPT

多人协作的好工具Git 常用指令：

最近也是用的比较多把，基本的操作也算是用得挺熟了，整理下思绪，以备不时之需。
最最基础的操作只稍微列下表：

  • git init 与 git clone
  • git add *
  • git commit -m “提交备注（一定要写详细 方便版本回退）”

  • git push [-u] origin master（github上默认版本库origin和分支master）
  • git pull origin master (如果上句用了-u 这句可以省略版本库和分支)
  • git fetch origin（获取） && git merge origin/master（合并） 这两句等价于上面那句 （通常这样比较安全）
  • git branch [-a] 查看本地分支 加-a可查看远程与本地对应情况
  • git branch -r -d origin/branch-name // 删除的本地对该远程分支的关
  • git checkout [-b] dev 切换分支（-b表示创建）并和远程库关联
  • git status 查看当前状态 只有工作区干净 才可以pull
  • git reflog 查看详细的更新历史
  • git reset —hard HEAD@{}//IDref2 回退版本

    下面列一些相对特殊点的用法：

    删除一个远程分支：

BFC与闭合浮动

date: 2015-08-10

BFC

  • 一个独立的渲染区域，只有Block-level box参与
  • 它规定了内部的Block-level Box如何布局，并且与这个区域外部毫不相干。

哪些元素会生成BFC?

  • 根元素
  • float属性不为none
  • position为absolute或fixed
  • display为inline-block, table-cell, table-caption, flex, inline-flex
  • overflow不为visible

BFC布局规则

  • 内部的Box会在垂直方向，一个接一个地放置。
  • Box垂直方向的距离由margin决定。属于同一个BFC的两个相邻Box的margin会发生重叠
  • 每个元素的margin box的左边， 与包含块border box的左边相接触(对于从左往右的格式化，否则相反)。即使存在浮动也是如此。
  • BFC的区域不会与float box重叠。
  • BFC就是页面上的一个隔离的独立容器，容器里面的子元素不会影响到外面的元素。反之也如此。
  • 计算BFC的高度时，浮动元素也参与计算

使父元素产生BFC即可闭合子元素的浮动

存在的问题：每种方式都会额外的改变一些东西：

  • float：为了让子元素闭合浮动，父元素也浮动起来了。
  • overflow：这里需要注意的一些问题，IE的滚动条是放在盒子模型的里面，而FF则放在外面，这样带来的问题是比较明显的。
  • display、position:这两个问题就更加明显了

Handlebarsjs

date: 2015-07-26

什么是Handlebarsjs?

  • Handlebars是一个Javascript模板引擎，轻松高效的编写语义化模板
  • 简单地说只要你定义一个模版，提供一个json对象，handlebars就能吧json对象放到你所给的模版中
  • 这样能将Web前端的视图和代码分离，降低两者之间耦合

如何告诉Handlebarsjs你要用哪个模板？

只需要这么一句就可以给Handlebarsjs传递一个模板
    var source   = $("#entry-template").html();

模板可以写在HTML的\标签中，也可以写在\标签中，分别如下: 

<body>中：   <任何标签x id="entry-template">模板信息</x>  
<script>中： <script id="entry-template" type="text/x-handlebars-template">模板信息</script>

Handlebarsjs如何使用模板？

获取模板后首先需要对模板进行编译：
    var template = Handlebars.compile(source);
然后通过 var result = temolate(Data);  （其中data是一个json对象）即可获得填好数据的DOM结构
最后通过$("#").before / .html / .after(result)    即可将结果渲染到指定DOM结构前/上/后

好了，基本用法就是这样了，下面就是在 “模板信息” 中深入了

Handlebars模板看起来就像是正常的Html，并使用了嵌入的 handlebars 表达式（以 双{ 开始，跟一些内容，以 双} 结束）。
