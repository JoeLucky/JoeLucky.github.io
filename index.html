javascript优化

date: 2015-10-13

脚本位置

JavaScript文件开始下载，会阻塞页面其他文件的下载，最佳脚本位置：
<body>
    <p>Hello world!</p>
    <!-- 以上所有用户可见的DOM结构结束 -->
    <script type="text/javascript" src="script1.js"></script>
    <script type="text/javascript" src="script2.js"></script>
    <script type="text/javascript" src="script3.js"></script>
</body>

组织脚本(内嵌、外链都一样要限制)

由于每个script标签初始下载时都会阻塞页面渲染，所以减少页面包含的script标签数量有助于改善这一情况。
浏览器在解析页面的过程中每遇到一个script标签，都会因执行脚本而导致延时，因此最小化延迟时间将会明显改善页面的总体性能。

对外链JavaScript文件，HTTP请求会带来额外的性能开销，单个100Kb优于5个20Kb文件。减少页面中外链脚本的数量将会改善性能。

无阻塞的脚本

延迟加载脚本

HTML 4 为script标签定义了一个扩展属性：defer。带有 defer 属性的script标签可以放置在文档的任何位置。
对应的 Js 文件将在页面解析到script标签时开始下载，但不会执行，直到 DOM 加载完成，即onload事件触发前才会被执行。
当一个带有 defer 属性的 Js 文件下载时，它不会阻塞浏览器的其他进程，因此这类文件可以与其他资源文件一起并行下载。

动态脚本元素

通过标准 DOM 函数创建<script>元素
var script = document.createElement ("script");
    script.type = "text/javascript";
    script.src = "script1.js";
    document.getElementsByTagName("head")[0].appendChild(script);

此文件当元素添加到页面之后立刻开始下载。重点在于：无论在何处启动下载，文件的下载和运行都不会阻塞其他页面处理过程。

压缩脚本

JavaScript是一种解释型语言,其所有的源代码都会被提交到客户端浏览器,因此源代码的体积会影响下载速度

Js语言本身层次方面

循环

  1. for(;;),while(),for(in)三种循环
    这三种循环中for(in)的效率极差，因为他需要查询散列键，只要可以就应该尽量少用。
  2. 如果要与数组的长度作比较，应该事先把数组的length属性放入一个局部变量中，减少查询次数。
  3. 局部变量和全局变量
    局部变量的速度要比全局变量的访问速度更快，因为全局变量其实是全局对象的成员，而局部变量是放在函数的栈当中的。
  4. 不使用Eval
    使用eval相当于在运行时再次调用解释引擎对内容进行运行，需要消耗大量时间。
  5. 少用嵌套对象成员（用局部变量代替）
    JavaScript的解释性，所以a.b.c.d.e，需要进行至少4次查询操作
  6. 类型转换
    • 数字转字符串 用””+1 性能上来说：(“” +) > String() > .toString() > new String()
    • 浮点数转整型 用Math.floor() 向下取整 /Math.round() /Math.ceil()
      Math是内部对象，所以Math.floor()其实并没有多少查询方法和调用的时间，速度是最快的。
  7. 字符串遍历操作
    对字符串进行循环操作，譬如替换、查找，应使用正则表达式;Js的循环比较慢，而正则表达式的操作是C语言写的API，性能好。
  8. DOM相关
    • 插入HTML,应指定容器后使用innerHTML，不用document.write
    • 创建DOM节点，不要使用字符串直接写HTML，无法保证代码的有效性字符串操作效率低,
      应该用document.createElement()，而如果文档中存在现成的样板节点/需要创建很多元素，应该是用cloneNode()，
      因为使用createElement()之后，你需要设置多次元素的属性，使用cloneNode()则可以减少属性的设置次数
    • 需多次重复返问同一DOM节点，应用变量缓存

BFC与浮动闭合

date: 2015-08-10

BFC

  • 一个独立的渲染区域，只有Block-level box参与
  • 它规定了内部的Block-level Box如何布局，并且与这个区域外部毫不相干。

哪些元素会生成BFC?

  • 根元素
  • float属性不为none
  • position为absolute或fixed
  • display为inline-block, table-cell, table-caption, flex, inline-flex
  • overflow不为visible

BFC布局规则

  • 内部的Box会在垂直方向，一个接一个地放置。
  • Box垂直方向的距离由margin决定。属于同一个BFC的两个相邻Box的margin会发生重叠
  • 每个元素的margin box的左边， 与包含块border box的左边相接触(对于从左往右的格式化，否则相反)。即使存在浮动也是如此。
  • BFC的区域不会与float box重叠。
  • BFC就是页面上的一个隔离的独立容器，容器里面的子元素不会影响到外面的元素。反之也如此。
  • 计算BFC的高度时，浮动元素也参与计算

使父元素产生BFC即可闭合子元素的浮动

存在的问题：每种方式都会额外的改变一些东西：

  • float：为了让子元素闭合浮动，父元素也浮动起来了。
  • overflow：这里需要注意的一些问题，IE的滚动条是放在盒子模型的里面，而FF则放在外面，这样带来的问题是比较明显的。
  • display、position:这两个问题就更加明显了

鉴于上述方式带来的额外影响，伪元素的方法某种程度上讲是更好的选择

什么是Handlebarsjs?

  • Handlebars是一个Javascript模板引擎，轻松高效的编写语义化模板
  • 简单地说只要你定义一个模版，提供一个json对象，handlebars就能吧json对象放到你所给的模版中
  • 这样能将Web前端的视图和代码分离，降低两者之间耦合

如何告诉Handlebarsjs你要用哪个模板？

只需要这么一句就可以给Handlebarsjs传递一个模板
    var source   = $("#entry-template").html();

模板可以写在HTML的\标签中，也可以写在\标签中，分别如下: 

<body>中：   <任何标签x id="entry-template">模板信息</x>  
<script>中： <script id="entry-template" type="text/x-handlebars-template">模板信息</script>

Handlebarsjs如何使用模板？

获取模板后首先需要对模板进行编译：
    var template = Handlebars.compile(source);
然后通过 var result = temolate(Data);  （其中data是一个json对象）即可获得填好数据的DOM结构
最后通过$("#").before / .html / .after(result)    即可将结果渲染到指定DOM结构前/上/后

正则表达式

date: 2015-10-10

之前研究了挺久，一段时间不用就忘记太多了，放上来备用！
[*（任意次） +（1/多次） ？（0/1次）]

判断邮箱：

/^([a-zA-Z0-9-])+@([a-zA-Z0-9-])+(.[a-zA-Z0-9_-])+$/
看起来简单 不过要注意@前必须是一个或者多个单词字符或者是-（这个-经常忘记）

判断中文字符：

/[^\u4E00-\u9FA5]/
匹配双字节字符(包括汉字在内)：[^x00-xff]

在正则表达式中使用 Unicode，必须使用\u开头，接着是字符编码的四位16进制表现形式
Unicode中文和特殊字符的编码范围
如：只能输入全角字符： /[^uFF00-uFFFF]/ [u前的/可以省略]

检查字符串是否包含连续重复的字母(后向引用)

/([a-zA-Z])\1/

衍生：/([a-z])([a-z])\2\1/ (可匹配abba)
说明：第一个小括号组名为1,第二个小括号组名为2
如果有嵌套的圆括号，顺序是按左括号的次序计算的

各种浏览器兼容问题

date: 2015-10-13

根据书籍、博客、实践等持续更新中……

要解决浏览器兼容问题，首先得知道当前使用的是什么浏览器

获取浏览器版本等信息: window.navigator.userAgent
如: “Mozilla/5.0 (X11; Linux i686) AppleWebKit/537.36 (KHTML, like Gecko) Chrome/44.0.2403.130 Safari/537.36”
各大浏览器内核需知: IE（Trident）; FF、Netscape（Gecko）; Safari、chrome(Webkit)
另外: Opera7以上抛弃了presto内核，加入Webkit内核大家族;最新的Blink内核……

当然也可以通过特定浏览器特有的属性或者方法来鉴别（更方便使用）

如IE9以下的有attachEvent（绑定事件），而一般服从w3c标准的使用addEventListener;
(本例的解决方案可以统一使用element.onclick = function(){}的用法，但是需注意赋值会覆盖而不会叠加事件)

以下分类分条列出各种兼容性问题

javascript

  1. window.event
    Question: IE有而FF没有
    Solve: var event = event || window.event; (此类常规解决方法以下省略不写)
  2. 事件源
    Question: IE(srcElement)、FF(target)
  3. 鼠标
    Question: IE(event.x,event.y) (event.offsetX,event.offsetY)[加滚动条]、
    FF(event.pageX,event.pageY) (event.layerX,event.layerY)[加滚动条]
    Solve: (event.clientX,event.clientY)[没滚动条时]
  4. 相关元素
    Question: IE(fromELement、toElement)[分别对应mouseover/out失去（获取）焦点、获得（失去）焦点元素]、
    FF(relatedTarget)[指另外一个元素，本身用this，如mouseover事件relatedTarget指失去焦点元素，this获得焦点]
    ……

  1. Question: 不同浏览器的标签默认的外补丁和内补丁不同
    Behavior: 随便写几个标签，不加样式控制的情况下，各自的margin 和padding差异较大。
    Solve: *{margin: 0;padding: 0;}
  2. Question: 图片默认有间距
    Behavior: 几个img标签放在一起的时候，有些浏览器会有默认的间距，加了通配符也不起作用。
    Solve: 使用float属性为img布局
