mysql相关 安装 百度的教程有图～ 基本操作 如何设置mysql允许远程登录

安装：sudo apt-get install mysql-server mysql-client（注意源的配置）
测试是否安装成功： sudo service mysql restart
1、mysql -u root -p   登录
2、显示所有的数据库：show databases; 显示库中所有的表：show tables;显示数据表的结构 describe 表名;
3、改变当前数据库为mysql：use mysql
4、修改属性为alter table testdata（表名）  modify id int  auto_increment;

5、远程登录mysql：mysql -h ip -u user -p
    如何设置mysql允许远程登录？
    1、vim /etc/mysql/my.cnf找到bind-address = 127.0.0.1 注释/修改指定IP
    2、重启 MySQL：sudo /etc/init.d/mysql restart
    3、授权用户能进行远程连接
        grant all privileges on *.* to root@"%" identified by "password" with grant option;
        flush privileges;
6、show variables like 'port';   // 查看端口号
7、linux下可视工具mysql-workbench (ubuntu下通过apt-get install直接安装)

nodejs与express相关 下载

1、安装任意版本的 node:  nvm
    wget -qO- https://raw.github.com/creationix/nvm/master/install.sh | sh
    nvm alias default 0.10.26
2、express    
    如果采用源码自编译安装nodejs，安装express时需要安装express-generator，
    如下所示：sudo npm install -g express-generator
3、npm install -d 安装依赖包
4、本地起服务器
    npm install -g http-server    
    项目目录下 http-server .     （这儿有个点～）
5、在对nodejs项目进行服务器迁移的时，在新的linux服务器上输入npm -v 命令后报 “cannot find module ‘npmlog’….”错误
原因是拷贝过去的bin目录下的npm文件并没有链接到module目录下的npm-cli.js文件
在执行 npm -v 命令时，路径问题导致npmlog模块无法找到。
    解决的方法有两个，
    第一种方式：删除bin目录下的npm文件，并用如下命令链接到npm-cli.js
    ln -s ../lib/node_modules/npm/bin/npm-cli.js
    第二种方式：将下载好的node-v0.10.33-linux-x64.tar.gz压缩包先放置在服务器上，
    再进行解压缩，此时npm是已经链接到npm-cli.js的。

screen相关 下载

1、干嘛用的：
    系统管理员经常需要SSH 或者telent 远程登录到Linux 服务器，经常运行一些需要很长时间才能完成的任务，比如系统备份、ftp 传输等等。通常情况下我们都是为每一个这样的任务开一个远程终端窗口，因为它们执行的时间太长了。必须等待它们执行完毕，在此期间不能关掉窗口或者断开连接，否则这个任务就会被杀掉，一切半途而废了。
    上面一段是复制的，简言之，screen最基本的功能是远程登录linux，启动程序X，关闭远程连接，程序X不停止。
2、基本用法：
    screen -S test1  新建
    Ctrl+a+d                    #按Ctrl+a，然后再按d即可保留当前的screen会话
    exit                        #这样就表示成功退出了
    ////////////////////////
    screen -ls       #查看所有screen会话  #假设显示 9649.test1   (Detached) Deatched表示没连接
    screen -r test1 (or 9649)
    如果上一条命令不能进入指定screen 如提示
    screen -d test1 将指定的screen作业离线
    screen -wipe  test1   #检查所有作业 删除已经无法使用的作业

Linux防火墙问题

有时候必须关闭防火墙才能操作某些东西 怎么关闭？如下
1) 重启后生效 
开启： chkconfig iptables on 
关闭： chkconfig iptables off 

2) 即时生效，重启后失效 
开启： service iptables start 
关闭： service iptables stop 

需要说明的是对于Linux下的其它服务都可以用以上命令执行开启和关闭操作。 

在开启了防火墙时，做如下设置，开启相关端口， 
修改/etc/sysconfig/iptables 文件，添加以下内容： 
-A RH-Firewall-1-INPUT -m state --state NEW -m tcp -p tcp --dport 80 -j ACCEPT 
-A RH-Firewall-1-INPUT -m state --state NEW -m tcp -p tcp --dport 22 -j ACCEPT
