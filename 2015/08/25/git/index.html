git常用操作

date: 2015-07-24

多人协作的好工具Git 常用指令：

最近也是用的比较多把，基本的操作也算是用得挺熟了，整理下思绪，以备不时之需。
最最基础的操作只稍微列下表：

  • git init 与 git clone
  • git add *
  • git commit -m “提交备注（一定要写详细 方便版本回退）”
  • git commit -a 相当于执行前面2句 弹出的备注不写会当作取消执行
  • git push [-u] origin master（github上默认版本库origin和分支master）
  • git pull origin master (如果上句用了-u 这句可以省略版本库和分支)
  • git fetch origin（获取） && git merge origin/master（合并） 这两句等价于上面那句 （通常这样比较安全）
  • git branch [-a] 查看本地分支 加-a可查看远程与本地对应情况
  • git branch -r -d origin/branch-name // 删除的本地对该远程分支的关
  • git checkout [-b] dev 切换分支（-b表示创建）并和远程库关联
  • git status 查看当前状态 只有工作区干净 才可以pull
  • git reflog 查看详细的更新历史
  • git reset —hard HEAD@{}//IDref2 回退版本

    下面列一些相对特殊点的用法：

    删除一个远程分支：

    git push origin :branch-name //将一个空分支push上去 == 删除远程分支 冒号前面是分支名（空）

    fork的项目副本怎么与源项目的更新保持同步

    什么个情况呢，就是人家的源项目(记作upstream)，你想做点贡献，必须fork到自己的远程库(记作origin) 然后clone下来(记作local)，修改后push到origin然后再pull request请求源项目合并，那么问题来了：
  • upstream是人家的项目，你当然无法控制他，这就是“协同开发” ，两种情况：
  1. 对upstream, fork、clone、修改、push 已经完成，此时upstream项目本身已经作了一些修改，此时点pull request，会提示无法自动合并，原因是版本落后了。
  2. 另外一个情况，upstream当前未修改过，pull request成功，但是upstream并没有merge这个pull request，而是自身进行了一些修改，那么这个已经产生的pull request将无效，无法merge，原因还是一样，版本落后了。
  • 以上两个情况都是local相对upstream版本落后无法合并，那么fork的项目副本怎么与源项目的更新保持同步？方法如下：
