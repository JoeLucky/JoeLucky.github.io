各种浏览器兼容问题

date: 2015-10-13

根据书籍、博客、实践等持续更新中……

要解决浏览器兼容问题，首先得知道当前使用的是什么浏览器

获取浏览器版本等信息: window.navigator.userAgent
如: “Mozilla/5.0 (X11; Linux i686) AppleWebKit/537.36 (KHTML, like Gecko) Chrome/44.0.2403.130 Safari/537.36”
各大浏览器内核需知: IE（Trident）; FF、Netscape（Gecko）; Safari、chrome(Webkit)
另外: Opera7以上抛弃了presto内核，加入Webkit内核大家族;最新的Blink内核……

当然也可以通过特定浏览器特有的属性或者方法来鉴别（更方便使用）

如IE9以下的有attachEvent（绑定事件），而一般服从w3c标准的使用addEventListener;
(本例的解决方案可以统一使用element.onclick = function(){}的用法，但是需注意赋值会覆盖而不会叠加事件)

以下分类分条列出各种兼容性问题

javascript

  1. window.event
    Question: IE有而FF没有
    Solve: var event = event || window.event; (此类常规解决方法以下省略不写)
  2. 事件源
    Question: IE(srcElement)、FF(target)
  3. 鼠标
    Question: IE(event.x,event.y) (event.offsetX,event.offsetY)[加滚动条]、
    FF(event.pageX,event.pageY) (event.layerX,event.layerY)[加滚动条]
    Solve: (event.clientX,event.clientY)[没滚动条时]
  4. 相关元素
    Question: IE(fromELement、toElement)[分别对应mouseover/out失去（获取）焦点、获得（失去）焦点元素]、
    FF(relatedTarget)[指另外一个元素，本身用this，如mouseover事件relatedTarget指失去焦点元素，this获得焦点]
    ……

html/css(这块了解得比较少)

  1. Question: 不同浏览器的标签默认的外补丁和内补丁不同
    Behavior: 随便写几个标签，不加样式控制的情况下，各自的margin 和padding差异较大。
    Solve: *{margin: 0;padding: 0;}
  2. Question: 图片默认有间距
    Behavior: 几个img标签放在一起的时候，有些浏览器会有默认的间距，加了通配符也不起作用。
    Solve: 使用float属性为img布局
